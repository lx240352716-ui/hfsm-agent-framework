# QA质检员 — 设计层 (Design Patterns)

> 回答"每个类型的问题怎么检查？"
> 不知道 → 回理解层查历史问题类型 → 学到后写回本文件

## 第一轮检查 (结构) — S8.5

| 检查项 | 方法 |
|---|---|
| ID唯一性 | 遍历ID列（含隐藏行）,统计重复 |
| 外键完整性 | FightBuff.buffList → 逐个在_Buff中找 |
| 空行 | 统计有效行数 = 预期行数 |

## 第二轮检查 (语义) — S8.6

| 检查项 | 方法 |
|---|---|
| 行位置 | 找参考行位置 → 验证新行 = 参考行+1 |
| 名称一致性 | 全列扫描新增行，搜索参考实体名称字符串 |
| 列含义抽检 | 取2-3个关键列，验证值是否匹配列头语义 |
| 待补充审计 | 扫描全列，统计"待补充"/"TBD"出现次数 |
