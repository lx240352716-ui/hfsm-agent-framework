# 战斗策划 — 例子层 (Examples)

> 翻译层遇到无法直接翻译的复杂需求时，到这里找拼接方案。
## 例子1：按阵营队友数给buff层数（大妈灵魂-第1句）

**需求**：大妈行动时，按场上大妈团成员存活数给自己对应层灵魂

**核心思路**：系统无法直接"数阵营队友"→ 用**标记buff做桥梁**，把"人数"转化成"buff数量"

### 步骤1：给阵营队友上标记（小ID先执行）

**FightBuff A**：

| 字段 | 值 | 说明 |
|------|---|------|
| timing | `21`（行动时） | 查速查表：行动时=21 |
| target | `102`（己方全体） | 102=己方全体，配合条件过滤 |
| buff条件 | `80007` | 筛选大妈团(camp=7) |
| buff概率 | `1002`（100%） | — |
| buffList | @标记buff | — |

**_Buff（标记）**：

| 字段 | 值 | 说明 |
|------|---|------|
| buff类型 | `1` | — |
| Buff因子 | `0` | 空标记 |
| 行动后生效计数 | `1` | 回合结束清掉 |
| id叠加的个数限制 | `1` | 每人只1层 |

**BuffActive（标记）**：

| 字段 | 值 |
|------|---|
| grade | 1 |
| buff参数1 | 0 |

### 步骤2：按标记数叠加灵魂（大ID后执行）

> **叠加通用结构**：用 `>N` 条件实现叠加，不是一次性给N层。
> 当标记=3时，>0、>1、>2 三个条件都满足，各加1层 = 3层。

**FightBuff B1-B4**（4个，条件分别为 >0, >1, >2, >3）：

| FightBuff | timing | target | buff条件 | buffList | 备注 |
|-----------|--------|--------|---------|----------|------|
| @FB_加1 | `21` | `101`(自己) | @条件_标记>0 | @灵魂buff | 有队友就+1 |
| @FB_加2 | `21` | `101` | @条件_标记>1 | @灵魂buff | 2人以上+1 |
| @FB_加3 | `21` | `101` | @条件_标记>2 | @灵魂buff | 3人以上+1 |
| @FB_加4 | `21` | `101` | @条件_标记>3 | @灵魂buff | 4人以上+1 |

概率/buffGradeList 均为 `1002` / `1`。

**_BuffCondition（检查标记数量）**：

| 条件 | 条件参数 |
|------|---------|
| @条件_标记>0 | `src1:self,type1:TeamTotalBuffNum,param1:@标记buff,op:more,src2:const,type2:TeamTotalBuffNum,const_val:0` |
| @条件_标记>1 | 同上，`const_val:1` |
| @条件_标记>2 | 同上，`const_val:2` |
| @条件_标记>3 | 同上，`const_val:3` |

> 参考：id=31350（TeamTotalBuffNum条件）

**_Buff（灵魂）**：

| 字段 | 值 | 说明 |
|------|---|------|
| buff类型 | `1` | — |
| Buff因子 | `0` | 纯标记/计数用 |
| id叠加的个数限制 | `99` | 未明确上限，默认99 |

**BuffActive（灵魂）**：

| 字段 | 值 |
|------|---|
| grade | 1 |
| buff参数1 | 0 |

**关键约束**：
- FightBuff A 的ID必须 < B1-B4 的ID（保证先打标再判断）
- 标记buff临时（行动后计数=1），不会累积
- 叠加用 `>N` 条件（不是 buffGradeList），每个条件各加1层
- 灵魂buff可叠加（后续第4句用countBuffIdAddBuff检查层数）

### 通用模式：按数量叠加

当需求说"按X数量给Y层buff"时，用 `>0, >1, >2, ...` 条件各加1层：

```
数量=1 → 满足>0 → 加1层
数量=2 → 满足>0,>1 → 加2层
数量=3 → 满足>0,>1,>2 → 加3层
数量=N → 满足>0到>N-1 → 加N层
```

## 例子2：属性降低 + 暴击计数 → 削HP（破绽 3100020-3100023）

**需求摘要**：命中降爆伤50%+2回合，被暴击2次削6%HP，上限30%

**拼接方案**：
```
FightBuff(命中) → 主buff(属性修改) → 关联(计数标记+计数器) → 触发(削HP)
```

| buff | buff类型 | Buff因子 | 关键字段 | BuffActive.p1 |
|------|---------|----------|---------|--------------|
| 主buff | 1 | `cridamage` | 取百分比=1, id叠加=1, 关联=标记;计数器 | ±百分比 |
| 计数标记 | 1 | `0` | 行动后计数=同主buff, 叠加不限 | 0 |
| 计数器 | 7 | `countBuffIdAddBuff` | 特殊配置=`标记ID,阈值&&结果ID,grade` | 1 |
| 削HP | 7 | `DelPercentHp` | 整场限制=N(控制上限) | 百分比 |

**FightBuff**：
| 触发 | 时机 | 目标 |
|------|------|------|
| 命中施加 | 7 | 1(技能目标) |
| 被暴击叠层 | 10 | 101(自己) |

**关键约束**：
- 主buff和计数标记**同生命周期**（行动后计数一致）
- 计数器的特殊配置引用标记ID → 需要先分配ID再填
