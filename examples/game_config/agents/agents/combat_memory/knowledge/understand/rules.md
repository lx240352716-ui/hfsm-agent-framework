# 战斗策划 — 理解层 (Understanding)

> 回答"这是什么？参考谁？核心要素？"
> 不知道 → 向用户确认 or 搜表自学 → 学到后写回本文件
> **核心原则：不确定就问，不要猜，分步确认后再设计**

## 需求拆解方法论（最高优先级）

> 任何被动技能/buff需求，必须先走完此拆解流程再进入设计。

### 第一步：分句

把需求原文拆成**独立的句子**，每句话 = 一条技能逻辑。一段需求通常2-5句话。

### 第二步：四要素树状拆解

每句话按以下树状结构拆分（一个角色+一个时机为树根，可有多个条件→事件分支）：

```
谁：[角色]
时机：[什么时候触发]
  ├─ 条件1：[前置判断]
  │  └─ 事件：[类型] + [目标] + [数量/数值] + [附加限制]
  ├─ 条件2：...
  │  └─ 事件：...
  └─ ...
```

**事件必须结构化**，不能是叙述性描述。每个事件拆为：
- **事件类型**：上buff / 扣血 / 回复 / 状态升级 / 清除buff ...
- **目标**：自己 / 命中目标 / 敌方全体 / 特定状态的目标 ...
- **数量/数值**：固定值 / 按条件动态计算（如按阵营队友数）
- **附加限制**：持续时间 / 可否驱散 / 仅PvP / 上限 ...

### 第三步：串联关系

画出句子之间的依赖：
- 哪条链产生状态 → 哪条链依赖该状态
- 是否存在循环（A产出→B消耗→D回收→A）
- 同生命周期约束（主buff消失时关联buff是否一起消失）

### 第四步：识别不确定项

拆完后，标记所有**找不到配置映射**的节点。这些就是需要向用户逐条确认的问题——**分步提问，不要一次性猜方案**。

### 示例：破绽（2句话）

```
第1句：
谁：自己
时机：每回合首个技能命中时
  └─ 条件：无
     └─ 事件：上buff | 目标：敌方目标 | 效果：抗爆伤-50% | 持续2回合 | 可驱散
              附带关联buff：暴击计数标记（同生命周期）

第2句：
谁：目标（持有破绽）
时机：被技能暴击时
  └─ 条件：暴击计数 ≥ 2次
     └─ 事件：上buff | 目标：自己 | 效果：削6%最大HP | 整场上限5次(30%)
```

### 示例：大妈灵魂（4句话）

```
第1句：
谁：大妈
时机：大妈行动时
  └─ 条件：无
     └─ 事件：上buff | 目标：自己 | 数量：按场上大妈团成员存活数

第2句：
谁：大妈
时机：释放技能时
  ├─ 条件1：消耗1层灵魂
  │  └─ 事件：上buff | 目标：命中的敌方 | 效果：灵魂抽取(治疗-10%) | 可驱散
  └─ 条件2：目标灵魂抽取 ≥ 5次
     └─ 事件：状态升级 | 目标：该敌方 | 从灵魂抽取→灵魂剥离(禁疗+受伤害+30%) | 2回合 | 不可驱散

第3句：
谁：大妈
时机：每回合行动后
  └─ 条件：敌方存在持有灵魂抽取或灵魂剥离的角色
     ├─ 事件1：上buff | 目标：自己 | 数量：按有状态的敌方数（回收灵魂）
     ├─ 事件2：扣血 | 目标：有状态的敌方 | 数值：10%当前HP | 仅PvP
     └─ 事件3：回复 | 目标：自己 | 数值：等于事件2扣除量

第4句：
谁：大妈
时机：灵魂层数变化时
  ├─ 条件1：≥3层 → 事件：上buff | 目标：自己 | 效果：最大生命+10%
  ├─ 条件2：≥8层 → 事件：上buff | 目标：自己 | 效果：所有伤害+15%
  └─ 条件3：≥15层 → 事件：上buff | 目标：自己 | 效果：属伤可暴击

串联：获取(1) → 消耗施加(2) → 状态升级(2) → 汲取回收(3) → 阶梯(4) ← 回流灵魂
```
