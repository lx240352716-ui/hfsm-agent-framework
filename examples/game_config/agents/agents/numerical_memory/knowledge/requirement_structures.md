# 需求结构映射

> 定义常见需求类型的表层级关系。LLM 在 split/locate 阶段查此文件，确定需要哪些表、表之间什么隶属关系。

---

## 礼包类

> 关键词：礼包、礼盒、限定礼包、节日礼包

### 结构

```
📦 礼包 Item（大类型=1001）
 └── _DropGroup × N（每组对应一种掉落物）
      └── 掉落物 Item（已有道具的 ID）

🏪 _ShopItem（商城上架配置）
```

### 关键字段关联

| 父表 | 字段 | 指向 | 说明 |
|------|------|------|------|
| Item | resourceDes | _DropGroup ID（逗号拼接） | 礼包打开后触发的掉落组 |
| _DropGroup | param | 掉落物 Item ID | 该掉落组实际掉什么道具 |
| _ShopItem | itemInfo | 礼包 Item ID | 商城卖的是哪个礼包 |

### 确认展示模板

fill 阶段必须按以下树形格式展示给用户确认：

```
📦 [礼包名] (Item, 参考 [ref_id])
 ├── 掉落组1 (_DropGroup, 参考 [ref_id]) → [掉落物名] ([掉落物ID] ✓已存在/⚠️需新增)
 ├── 掉落组2 (_DropGroup, 参考 [ref_id]) → [掉落物名] ([掉落物ID] ✓已存在/⚠️需新增)
 └── 掉落组N ...

🏪 商城上架 (_ShopItem, 参考 [ref_id])
    售价: [金额] / 限购: [次数]
```

### 注意事项

- 掉落组数量 = 礼包内含道具种类数，需与用户确认
- 掉落物 Item 通常已存在，需查源表验证并展示名字
- 如掉落物不存在，需标记 ⚠️ 并额外新增 Item 行

## 装备类

> 关键词：装备、武器、铠甲、饰品

### 结构

```
Equipment（装备主表）
 └── BuffActive × N（装备附带的 buff 效果）
```

### 关键字段关联

| 父表 | 字段 | 指向 | 说明 |
|------|------|------|------|
| Equipment | buff | BuffActive ID | 装备附带的 buff |

> ⚠️ 装备类结构待实际需求验证后补充完善。

---

## 通用规则

1. **树形展示**：fill 确认时，所有需求类型都必须按隶属关系用树形格式展示
2. **叶子节点标状态**：已存在的实体标 `✓已存在`，需新增的标 `⚠️需新增`
3. **数量确认**：子行数量（如掉落组个数）必须展示并等用户确认
