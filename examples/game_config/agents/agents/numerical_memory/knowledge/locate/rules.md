# locate 阶段规则

> 定位表 + 找参考行。hook 搜索，LLM 选择。

## 字段过滤规则

> 从 table_directory.md 拿到完整字段后，通过查 **同类参考数据** 过滤出用户需要填的字段。

### 过滤步骤

1. **查同类数据**：`query_db("SELECT * FROM [表名] LIMIT 5")` 取至少 3 行参考
2. **标记固定值**：某字段所有行值相同 → 固定值，注明默认值，不让用户填
3. **标记默认值**：某字段全为空 / 0 / None → 默认值，不展示
4. **主键分配**：主键字段（通常第一列）→ 用 `max_id` 自动分配，不让用户填
5. **剩余字段** = 需用户决策 → 展示给用户

### 输出格式

每个模块的字段分为三类：
- `fixed`：固定值（如礼包的 物品大类型=1001），自动填入
- `auto`：自动分配值（如主键ID），自动填入
- `input`：需要用户决策的字段 → **只展示这类**

## 参考行定位方式

对每个模块，你需要输出 **表名 + 参考行定位方式（三选一）**：

### 1. 用户明确提到了参考
- 如 "参考觉醒徽章礼盒" → 输出 `search_keywords: ["觉醒徽章", "礼盒"]`
- 如 "商城参考78" → 直接输出 `_ref_id: "78"`

### 2. 可从关联表推理
- 如 Item 参考 619021，它的 params 字段里有 _DropGroup 的 ID
- → 直接输出 _DropGroup 的 `_ref_id`（从 Item 的数据推理）

### 3. 都不确定
- 输出 `search_keywords: ["需求关键词"]`
- hook 会搜源表返回候选，fill 阶段再选

## locate_result.json 输出格式

```json
{
  "modules": [
    {
      "name": "模块名",
      "table": "真实表名",
      "_ref_id": "已知ID",
      "search_keywords": ["关键词"],
      "fields": [
        { "cn": "中文名", "en": "英文名", "type": "fixed/auto/input" }
      ]
    }
  ]
}
```

> 每个模块必须包含 `table` + `_ref_id` 或 `search_keywords` 之一。
> ⚠️ **铁规：每个模块最终必须确定 `_ref_id`，不支持从零建行。** 如果 search_keywords 搜不到合适的，要求用户提供参考 ID。
