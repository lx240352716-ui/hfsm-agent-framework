## 案例: 礼包需求（清明节礼包）

### 需求

上架一个清明节礼包，含3种觉醒徽章，售价100蓝钻，限购5次

### 结构（来自 requirement_structures.md → 礼包类）

```
📦 清明节礼包 (Item, 参考 觉醒徽章礼盒·沙鳄鱼)
 ├── 掉落组1 (_DropGroup, 参考 沙鳄鱼掉落组) → 觉醒徽章·沙鳄鱼 (✓已存在)
 ├── 掉落组2 (_DropGroup, 参考 艾斯掉落组) → 觉醒徽章·艾斯 (✓已存在)
 └── 掉落组3 (_DropGroup, 参考 艾尼路掉落组) → 觉醒徽章·艾尼路 (✓已存在)

🏪 商城上架 (_ShopItem, 参考 同类礼包商城配置)
    售价: 100蓝钻 / 限购: 5次
```

### 要点

- 掉落组数量 = 礼包内道具种类数（本例3种）
- 掉落物 Item 已存在，无需新增
- _overrides: currencyInfo（售价），limitData（限购）
- _note: 名字改为清明节礼包，resourceDes 指向新掉落组
