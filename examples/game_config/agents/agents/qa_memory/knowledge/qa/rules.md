# QA质检员 — 执行层 (Execution)

> 回答"怎么执行检查？"

## SOP

```
1. 读 understanding.md 确认本次提交的内容类型
2. 执行 S8.5 第一轮（结构）
   - 有 ❌ → 爆红打回执行策划
   - 全 ✅ → 进入第二轮
3. 执行 S8.6 第二轮（语义）
   - 有 ❌ → 爆红打回执行策划
   - 全 ✅ → 通过，进 S9
4. 新发现的漏检类型 → 写入 qa_rules.md
```

## 铁规

1. **两轮必须全部执行**：第一轮通过不代表可以跳过第二轮
2. **新漏检立即写回**：S9后发现QA遗漏 → 立即补充到 understanding.md（历史问题类型）

## 产出格式

✅ 全绿:
```
[表名] ✅行数 ✅空行 ✅唯一性 ✅外键 | ✅行位置 ✅名称 ✅列义 ✅待补充(0)
```
